## [0.45.0] ‚Äî 2026-06-28

### Changed
- **Audio node-churn eliminated** (`v0.45.0`, review X1 ‚Äî the GC-hitch suspect behind frame stutter in all three games). Every music voice used to build its full Web Audio chain **per note** (7‚Äì11 nodes: WaveShaper + BiquadFilters + Gains + oscillators); now every WaveShaper/BiquadFilter/Gain is built **once per (voice, track output)** and reused, and a note creates only its one-shot sources (oscillators / buffer sources ‚Äî unreusable by spec): bass 3, arp 2, lead 3, guitar 6, kick 2, hat 1. Envelopes and filter sweeps run as sequential automation on the persistent nodes (the lookahead scheduler emits ascending time), and pitched voices keep a **round-robin pool** (K=3 for bass/wob/arp/tesla, which the intensity layer doubles at the same t; K=2 for guitar/guitarLead/lead) so simultaneous notes and release tails never share an envelope. LFO/vibrato oscillators stay per-note so their phase still locks to note-on. **Critical catch the harness couldn't see:** `ensure()` wraps the gain-family factories to TTL-register note nodes and *auto-disconnect them after 2.5 s* ‚Äî persistent chains built inside the scheduler would have been severed 2.5 s into every track (silent in the mock, dead in a browser); `chainsFor` now suspends that tracking while building. The TTL registry still serves sfx (unchanged per-call chains). Files: `audio.js`, `audio-smoke.mjs` (+4 churn assertions proving ZERO heavy-node creation over 400 steady-state ticks incl. intensity chords + wobble LFO tracks, node-cleanup test rewritten to the new architecture incl. sfx register+sweep; test-wrapper restore fixed to reassign rather than `delete`, which was stripping audio.js's own instance wrappers).
