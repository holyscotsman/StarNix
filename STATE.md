## What StarNix is
A Nutanix **NCP-MCI** certification study game: a shell that boots a shared cinematic + a menu of three sub-games that all teach the same exam material via spaced retrieval. Enemy: the **BCM (Broad Communication Military)**. Crew: the **Nutanix Starlight Rescue Crew (NX-SRC)**, a.k.a. the **StarNix Rescue Crew**.
- **ARM** ‚Äî Acropolis Rescue Mission (2D flight + collect): rebuild the MCI station.
- **KBB** ‚Äî Kuiper Belt Battle (2D roguelike, Slay-the-Spire √ó Balatro): hunt the BCM warship.
- **CC** ‚Äî Chasm Chase (3D Temple-Run, Three.js): chase the escaped squadron down a canyon.

## Build model (now)
Plain-JS modules, **no build step**, each attaches to `window.StarNix`. The Core chat **concatenates** them into one `index.html` for Google Apps Script. (TypeScript/Vite/clasp deferred to containerization.) Three.js r128 loaded from cdnjs in `<head>` for CC.

**Commands** (run in the folder with the module files):
- `node build.mjs` ‚Üí regenerates `index.html` (order: core ‚Üí **assets** ‚Üí shell ‚Üí audio ‚Üí arm ‚Üí cc ‚Üí kbb ‚Üí boot).
- `node verify-build.mjs` ‚Üí headless jsdom integration check; must print `ALL GREEN`.
- `node audio-smoke.mjs` ‚Üí mock-AudioContext check of the synth path (jsdom has no AudioContext); must print `AUDIO SMOKE: ALL GREEN`.
- `node scheduler-test.mjs` ‚Üí unit-tests the Leitner scheduler; must print `SCHEDULER TEST: ALL GREEN`.
- `node multi-answer-test.mjs` ‚Üí multi-select schema/grading; must print `MULTI-ANSWER TEST: ALL GREEN`.
- `node shuffle-test.mjs` ‚Üí answer-order shuffle (learning integrity); must print `SHUFFLE TEST: ALL GREEN`.
- `node timer-test.mjs` ‚Üí dynamic per-question timer formula; must print `TIMER TEST: ALL GREEN`.
- `node import-questions.mjs` ‚Üí authoring `.md` ‚Üí `questions.js`; must print `VALIDATION: ALL GREEN`.
- **`audio-test.html`** ‚Üí open in a browser to actually *hear* the tracks/SFX (the only real audio check). Needs `audio.js` beside it.
- `node balance-probe.mjs` ‚Üí headless KBB balance probe (auto-plays runs at various accuracies, reports depth reached / min HP / coins / death cause). For tuning, not a pass/fail gate.

**Deploy to Apps Script:** project has `Code.gs` with `doGet(){ return HtmlService.createHtmlOutputFromFile('index'); }` + an HTML file named `index` containing `index.html`. Update = paste new `index.html` ‚Üí Deploy ‚Üí Manage deployments ‚Üí new version.

## The frozen contract (01 ¬ß9a) ‚Äî brief
- `window.StarNix = { core, shell, registerGame(m), getGame(id), registerAudio(impl), boot(root,opts), initCore(opts), makeContext(gameId) }`.
- A game = `{ id:"ARM"|"KBB"|"CC", mount(root, ctx), unmount() }`; `unmount` must leave **zero residue** (cancel RAF, remove listeners, free DOM).
- `ctx` (per game): **guaranteed** `questions, mastery, persistence, rng, audio, theme, telemetry`; **additive** `ai, settings, sanitize`; and `exit()` (added by the shell's `enterGame` so a game can request return-to-menu).
- Audio engine installs at `window.StarNix.core.audio` (audio.js). 5 tracks: `cinematic, menu, arm, kbb, cc`. The shell plays the right track on enter; each game also calls `playTrack` itself.
- Verification is **structural** (wired, no errors across frames, clean lifecycle). It does NOT prove visual correctness, that CC's WebGL renders, or that audio sounds right ‚Äî those are real-browser-only.

### CORE/audio ‚Äî fix music degradation over time (scheduler catch-up clamp) ‚Äî 2026-06-24 ‚Äî ‚úÖ build green (195/195) ¬∑ v0.6.16
- **Bug (Jason playtest):** music "starts to go slow and then sounds really weird after a bit of time."
